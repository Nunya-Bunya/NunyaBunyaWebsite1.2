Go-Live Checklist — Nunya Bunya Marketing Stack

Step 1: Get a VPS (5 minutes)

  • ☐ Sign up at Hetzner Cloud (or any VPS provider)
  • ☐ Create a server:
  • Type: CX41 (4 vCPU, 8GB RAM, 80GB SSD) — ~$15 AUD/month
  • OS: Ubuntu 24.04
  • Location: Choose nearest to your clients (Falkenstein/Nuremberg for EU, Hillsboro for US, or Singapore for APAC)
  • SSH Key: Add your Mac's public key (cat ~/.ssh/id_rsa.pub)
  • ☐ Note down the server IP: _

Step 2: Configure DNS (5 minutes)

Go to your domain registrar (Cloudflare, Namecheap, etc.) and add these A records:

SubdomainTypeValueProxy
mautic.nunyabunya.com.auAYOURSERVERIPDNS only (grey)
crm.nunyabunya.com.auAYOURSERVERIPDNS only (grey)
analytics.nunyabunya.com.auAYOURSERVERIPDNS only (grey)

  • ☐ DNS records created
  • ☐ Verify propagation: dig mautic.nunyabunya.com.au +short (should return server IP)

Step 5: Configure SMTP (5 minutes)

Choose ONE option and fill in the SMTP fields in .env:

Option A: Resend (fastest — you already have an account)

  1. Go to resend.com/domains
  2. Add and verify nunyabunya.com.au (add the DNS records they give you)
  3. Go to resend.com/api-keys — copy your API key
  4. Fill in .env:
   SMTP_HOST=smtp.resend.com
   SMTP_PORT=465
   SMTP_USER=resend
   SMTP_PASSWORD=re_YOUR_API_KEY

Option B: Amazon SES (cheapest for volume)

  1. Sign up at aws.amazon.com
  2. Go to SES > Verified Identities > Create identity > Domain > nunyabunya.com.au
  3. Add the DKIM/SPF DNS records
  4. Go to SES > SMTP Settings > Create SMTP credentials
  5. Fill in .env:
   SMTP_HOST=email-smtp.ap-southeast-2.amazonaws.com
   SMTP_PORT=587
   SMTP_USER=YOUR_SES_SMTP_USER
   SMTP_PASSWORD=YOUR_SES_SMTP_PASSWORD
  • ☐ SMTP configured in .env
  • ☐ Domain verified with email provider

Step 6: Fill In Supabase Connection (2 minutes)

  1. Go to Supabase Dashboard > Your project > Settings > Database
  2. Copy the "Connection string" (URI format)
  3. Fill in .env:
   SUPABASE_DIRECT_URL=postgresql://postgres:YOUR_PASSWORD@db.fsxxucjnpjpewvslmnbj.supabase.co:5432/postgres
  • ☐ Supabase connection string added to .env

Step 8: Complete Setup Wizards (20 minutes)

Mautic — https://mautic.nunyabunya.com.au

  1. Complete the setup wizard (DB is pre-configured)
  2. Create admin account (email: ben@nunyabunya.com)
  3. Settings > Configuration > Email Settings > test email
  4. Settings > Configuration > API Settings > Enable API = Yes
  5. Settings > API Credentials > New > Basic Auth
  6. Copy the API username and password → fill in .env:
   MAUTIC_API_USER=your_api_username
   MAUTIC_API_PASSWORD=your_api_password
  • ☐ Mautic configured
  • ☐ API credentials saved to .env

Twenty CRM — https://crm.nunyabunya.com.au

  1. Create workspace and admin account
  2. Set up pipeline stages: Lead → Qualified → Proposal → Won → Lost
  3. Settings > Developers > API Keys > Create new key
  4. Copy the API key → fill in .env:
   TWENTY_API_KEY=your_api_key
  • ☐ Twenty CRM configured
  • ☐ API key saved to .env

Matomo — https://analytics.nunyabunya.com.au

  1. Complete setup wizard
  2. Add first website: nunyabunya.com.au
  3. Copy the JavaScript tracking code (you'll add it to websites later)
  4. Settings > Personal > Security > Create new auth token
  5. Copy the token → fill in .env:
   MATOMO_AUTH_TOKEN=your_token
  • ☐ Matomo configured
  • ☐ Auth token saved to .env
  • ☐ Tracking JS snippet saved for later

Metabase — https://reports.nunyabunya.com.au

  1. Create admin account
  2. Add data sources:
  • Supabase: PostgreSQL, use your SUPABASEDIRECTURL details
  • MySQL (Mautic): Host: nb-mysql, Port: 3306, User: metabasero, Password: same as MYSQLROOTPASSWORD, DB: mauticdb
  • MySQL (Matomo): Same host/user, DB: matomo_db
  • MySQL (Shlink): Same host/user, DB: shlink_db
  • Postgres (Twenty): Host: nb-twenty-db, Port: 5432, User/Pass from .env, DB: twenty
  1. Admin > Settings > Embedding in other Applications > Enable
  2. Copy the secret key → fill in .env:
   METABASE_SECRET_KEY=your_secret_key
   METABASE_ADMIN_PASSWORD=whatever_you_set
  • ☐ Metabase configured
  • ☐ All 5 data sources connected
  • ☐ Embedding enabled, secret key saved to .env

Shlink — https://links.nunyabunya.com.au

  1. Open the Shlink web client
  2. Add server: URL = https://go.nunyabunya.com.au, API Key = your SHLINKAPIKEY from .env
  3. Create a test short URL to verify
  • ☐ Shlink web client configured
  • ☐ Test short URL working

Step 12: Add Matomo Tracking to Websites (10 minutes)

Add the Matomo JavaScript snippet to each client's website <head>:

<!-- Matomo -->
<script>
  var _paq = window._paq = window._paq || [];
  _paq.push(['trackPageView']);
  _paq.push(['enableLinkTracking']);
  (function() {
    var u="https://analytics.nunyabunya.com.au/";
    _paq.push(['setTrackerUrl', u+'matomo.php']);
    _paq.push(['setSiteId', 'SITE_ID_HERE']);
    var d=document, g=d.createElement('script'), s=d.getElementsByTagName('script')[0];
    g.async=true; g.src=u+'matomo.js'; s.parentNode.insertBefore(g,s);
  })();
</script>
<!-- End Matomo -->
  • ☐ Tracking added to nunyabunya.com.au
  • ☐ Tracking added to other client websites (as they onboard)
