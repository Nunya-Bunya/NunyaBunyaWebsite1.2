How This Works

Every agent has a Directive (prompt spec in /Directives/) and an Execution script (Python in /Execution/). Agents are organized into Workflow Pipelines — sequences of agents that run together to produce a deliverable. Each client gets assigned the pipelines relevant to their service package.

Workflow Pipelines

These are the standard sequences agents run in to produce deliverables. Each pipeline = one SOP.

Pipeline 1: New Client Onboarding

Strategy Agent → Foundation Package Builder → Knowledge Base → Content Marketing Agent → SEO Opportunity Finder → Social Media Agent (setup) → Reporting Agent (setup)

Trigger: New client signs

Output: Complete client setup — strategy, brand docs, knowledge base, content plan, SEO baseline, social accounts, reporting dashboard

SOP: SOPs/Client Onboarding/

Pipeline 2: Weekly Content Production

Content Generator → Social Media Agent → Short Form Video Agent → Content Machine (overflow)

Trigger: Weekly (Monday)

Output: Blog post, social posts (all platforms), video scripts, scheduled posts

SOP: SOPs/Content Production/

Pipeline 3: Weekly SEO Cycle

SEO Agent → Local SEO Tool (if needed) → Content Machine (SEO articles)

Trigger: Weekly (Tuesday)

Output: SEO report, new keyword targets, landing pages, optimized articles

SOP: SOPs/SEO/

Pipeline 4: Ads Monitoring & Optimization

Ads Optimization Agent → (flagged to team) → Content Generator (new ad copy if needed)

Trigger: Daily (automated) + Weekly review

Output: Performance flags, new ad copy variants, budget recommendations

SOP: SOPs/Ads Management/

Pipeline 5: Reputation Monitoring & GBP Analytics

GBP Metrics Collector → Reputation Agent → (negative alerts to team) → Knowledge Base (log interactions)

Trigger: Daily (automated — GBP at 6 AM AEST, Reputation every 4 hours)

Output: GBP daily metrics (impressions, actions, keywords, posts, media), auto-responses to positive reviews, draft responses for negative, alerts

SOP: SOPs/Reputation Management/

Pipeline 6: Weekly Client Reporting

Reporting Agent → Knowledge Base (log report)

Trigger: Weekly (Friday)

Output: Client-ready HTML/PDF report with insights and recommendations

SOP: SOPs/Reporting/

Pipeline 7: Lead Generation Sprint

Google Maps Lead Gen / SERP Scraper / Apify Scraper → Lead Qualifier → Create Proposal → Instantly Autoreply

Trigger: On-demand or scheduled

Output: Qualified lead list, proposals, automated follow-up

SOP: SOPs/Lead Generation/

Pipeline 8: New Business Acquisition (Nunya Bunya Internal)

Upwork Scrape & Apply → Create Proposal → First Client Package

Trigger: On-demand

Output: Job applications, proposals, full client acquisition package

SOP: SOPs/Sales & Proposals/

Pipeline 9: Website Redesign + Case Study (Productized Service)

Phase 1: Audit & Capture → Phase 2: Content & Messaging → Phase 3: AI Photoshoot → Phase 4: Website Build → Phase 5: Case Study Kit → Phase 6: Upsells

Trigger: python websiteredesignagent.py --client <client_id> --headshot <path>

Output: Complete redesigned website, AI photography, case study video script + shot list, AI receptionist config, bespoke photoshoot brief

Phases can run independently: --phase 1 through --phase 6

Upsells: AI Receptionist ($149/mo), Bespoke Photoshoot Brief ($800-1500), Case Study Video (free — NB keeps content)

Scaling Notes (100-Client Architecture)

Current Capacity

  • Each agent is parameterized by client_id — no code changes needed per client
  • Google Sheets, Supabase, and GoHighLevel are the shared data layers
  • Buffer handles social scheduling across all clients

To Scale to 100 Clients

  1. Client Config Store — Each client gets a JSON config with all IDs (Google Ads, GHL, GBP, etc.)
  2. Scheduler — Cron-based pipeline runner: runpipeline(clientid, pipeline_name)
  3. Queue System — Rate-limit API calls across clients (Google, Meta, Buffer limits)
  4. Dashboard — Central status board: which pipelines ran, which failed, which need review
  5. Team Approval Queue — All ad copy, negative review responses, and proposals route through approval before going live
  6. Client Tiers — Map service packages to pipeline assignments:
  • Starter: Content + SEO + Reputation
  • Growth: Content + SEO + Ads + Reputation + Reporting
  • Full Service: All pipelines
