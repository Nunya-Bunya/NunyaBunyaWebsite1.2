Operations

Delivery Model

  • Contractor-based, no employees. Ben reviews daily, not doing execution work.
  • AI-agent-driven production: 43 agents handle content generation, SEO, ads, reporting, research
  • Human-in-the-loop: all client-facing deliverables require review before publishing

New Client Onboarding (5 phases, 1-2 hours total)

  1. Stack Config (30min): Slack channel, client_configs.json, Mautic segment + drip, Matomo site, Twenty CRM entry
  2. Landing Page (30min): Offer page + lead magnet page + thank-you page, wire forms to Mautic + Slack
  3. Tracking (15min): Meta Pixel + Matomo tracking on all pages, Shlink short links
  4. Ad Setup (30min): Ad copy variations, retargeting audiences, campaigns
  5. Handoff (15min): Welcome email, Metabase dashboard, update internal tracking

Standard Funnel Architecture

Traffic --> Landing Page --> Form Submit --> (Mautic contact + Slack notification + thank-you page) --> 5-email drip sequence --> Conversion --> Post-conversion automation (review request day 7)

Lead Scoring (Mautic)

Email opened +5 | Link clicked +10 | Landing page visit +10 | Form submit +20 | Pricing page +15 | Multiple visits in 7 days +10 | Hot lead alert at 50+ points --> Slack alert + "Hot Leads" segment

Automation Recipes (n8n + Mautic)

  1. New lead --> Mautic contact + Slack notification + CRM entry
  2. Hot lead (50+ points) --> Urgent Slack alert + "Hot Leads" segment
  3. Follow-up reminder (48hrs, no booking) --> Slack reminder
  4. Review request (7 days post-conversion) --> Email with Google review link
  5. Re-engagement (30 days inactive) --> "We miss you" email, then archive if still inactive

Offboarding

Full 10-phase process documented: notice handling, continued service during 30-day notice, platform-by-platform access revocation, creative/data handover, financial close-out, exit survey, case study request, review request, alumni referral program ($250 per referred paying client), relationship maintenance at 1/3/6 months post-departure.

Strategy

Current Phase: Pre-Revenue / Infrastructure Complete

The hard part (building the full self-hosted marketing stack + 43 AI agents + 8 pipelines) is done. The work now is getting the first paying client and proving the system works end-to-end with a real human.

Immediate Priorities

  1. First paying client -- even at a discount, even as a trade deal. One real client with real results > a perfect system with zero users.
  2. Build the Launchpad orchestrator -- single command to run the full onboarding package
  3. Build Slack-based approval queue -- review all AI-generated deliverables before they go live
  4. Document the before/after obsessively -- this becomes the case study that sells the next 5 clients

Growth Plan

  1. Prove with 5-10 clients, document failure modes
  2. Build NBHQ dashboard approval queue (replaces Slack review once at 3+ clients)
  3. Scale to 13 clients / $35K MRR by mid-2027
  4. THEN consider self-serve SaaS product (different business, $10M+ idea if executed)

Competitive Advantage

Nobody offers the full stack (website + landing page + ads + lead magnet + email + booking + automation + content + brand strategy) generated from a single brief and deployed automatically. GoHighLevel, Vendasta, Durable.ai, Jasper -- they each do pieces. NB does the whole thing, self-hosted, for $16/mo in infrastructure.

Key Gaps to Close

  1. Website Builder Agent -- biggest missing piece. No agent yet takes scraped content + stock images and outputs a deployable website
  2. Approval Queue System -- critical for quality control at scale. "If a human will see it, you see it first."
  3. Cal.com client provisioning -- not yet automated per-client
  4. Stripe integration -- payment links not yet wired through n8n
  5. Stock image sourcing pipeline -- needs Unsplash/Pexels API alongside Replicate Flux

Core Operating Principles

  • "If a human will see it, you see it first." — Every client-facing or public-facing deliverable must be reviewed and approved by Ben before deployment. AI generates drafts. Humans make them premium. Infrastructure (tracking, segments, CRM entries, Slack channels) can auto-deploy. Content, ads, emails, websites, and designs cannot.
  • "The workbench is the product." — The agents are the factory floor. The review/editing workbench (where Ben edits, reworks, and approves) is what makes NB's output premium. Prioritize the review experience over adding more agents.
  • "The approval queue is the product." — Quality control isn't overhead. It IS the differentiator between generic AI output and premium agency output.
