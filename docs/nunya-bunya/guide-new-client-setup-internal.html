New Client Setup Guide

Internal SOP -- Nunya Bunya

Purpose: Step-by-step process for onboarding a new client into the Nunya Bunya marketing stack. Follow this exactly for every new client.

Time to complete: 1-2 hours per client.

Prerequisites

Before starting, you need:

  • ☐ Signed client contract
  • ☐ Client's business name, website URL, and logo
  • ☐ Client's brand colours (hex codes)
  • ☐ Access to client's domain DNS (or their registrar login)
  • ☐ Client's social media account access (or login credentials)
  • ☐ Understanding of their target audience and service offerings
  • ☐ Agreement on which package/services they're getting

Phase 1: Stack Configuration (30 minutes)

1.1 Create Slack Channel

  1. Go to the NunyaBunya Slack workspace
  2. Create a new channel: #client-name (lowercase, hyphens)
  3. Set the channel topic: "Client Name -- [package name] -- marketing + reporting"
  4. Pin a message with key client info: website, contact person, package, start date

1.3 Create Mautic Segment

Option A: Via API (preferred)

TOKEN=$(curl -s -X POST 'https://mautic.nunyabunya.com/oauth/v2/token' \
  -d 'grant_type=client_credentials' \
  -d 'client_id=YOUR_OAUTH_CLIENT_ID' \
  -d 'client_secret=YOUR_OAUTH_CLIENT_SECRET' | python3 -c "import sys,json;print(json.load(sys.stdin)['access_token'])")

curl -s -X POST 'https://mautic.nunyabunya.com/api/segments/new' \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Client Name - All Contacts",
    "alias": "client-name-contacts",
    "isPublished": true,
    "isGlobal": true
  }'

Option B: Via Mautic UI

  1. Go to https://mautic.nunyabunya.com
  2. Segments > New
  3. Name: "Client Name - All Contacts"
  4. Save

Note the segment ID and update client_configs.json.

1.4 Create Mautic Email Sequence

Look up the client's businesstype in dripsequences.json to find their email templates. Then create each email via API:

curl -s -X POST 'https://mautic.nunyabunya.com/api/emails/new' \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Client Name - Email 1: Welcome",
    "subject": "Subject line here",
    "customHtml": "<html>Email body here</html>",
    "emailType": "template",
    "isPublished": true,
    "fromName": "Client Name"
  }'

Repeat for each email in the drip sequence. Note all email IDs.

1.6 Add Client to Matomo

  1. Go to https://analytics.nunyabunya.com
  2. Settings > Websites > Add a new website
  3. Enter client's website URL
  4. Note the Site ID
  5. Update clientconfigs.json with matomosite_id

1.7 Add Client to Twenty CRM

  1. Go to https://crm.nunyabunya.com
  2. Create a new Company for the client
  3. Add the contact person as a Person linked to the Company
  4. Note the company ID and update client_configs.json

Phase 2: Landing Page Setup (30 minutes)

2.1 Create Landing Page

Use the landing page template at /01-Nunya Bunya/Site/nunyabunya-site/landing.html as a base.

For each client, create:

  1. Offer landing page -- the main offer (free audit, discount, etc.)
  2. Lead magnet page -- downloadable resource in exchange for email

Each landing page needs:

  • Client's branding (colours, logo, fonts)
  • Clear headline and value proposition
  • Form with fields: Name, Email, Phone (optional), and any qualifying questions
  • Meta Pixel tracking code (use the Nunya Bunya pixel: 251483972902519)
  • Matomo tracking code (use the client's Site ID)
  • Form submission handler that:
  • Creates contact in Mautic with client tag
  • Sends Slack notification to client channel
  • Redirects to thank-you page

2.2 Form Submission Handler

Create an API route (e.g., /api/lead.js for Vercel) that does three things on form submit:

// 1. Create contact in Mautic
const mauticResponse = await fetch('https://mautic.nunyabunya.com/api/contacts/new', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
    'Authorization': 'Basic ' + Buffer.from('user:pass').toString('base64')
  },
  body: JSON.stringify({
    firstname: data.name,
    email: data.email,
    phone: data.phone,
    tags: ['client-name', 'landing-page']
  })
});

// 2. Send Slack notification
await fetch('https://slack.com/api/chat.postMessage', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
    'Authorization': 'Bearer ' + process.env.SLACK_BOT_TOKEN
  },
  body: JSON.stringify({
    channel: 'CLIENT_SLACK_CHANNEL_ID',
    text: `New lead from Client Name!\nName: ${data.name}\nEmail: ${data.email}\nPhone: ${data.phone}`
  })
});

// 3. Return success (redirect to thank-you page on client side)
return Response.json({ success: true });

2.3 Thank-You Page

Create a thank-you page that:

  • Confirms their submission
  • Sets expectations ("We'll be in touch within 24 hours")
  • Fires the Meta Pixel Lead event: fbq('track', 'Lead');
  • Optionally offers a next step (book a call, download resource)

2.4 Deploy

cd /path/to/client-site
npx vercel --prod

2.5 Test End-to-End

Submit a test form and verify:

  • ☐ Contact appears in Mautic with correct tags
  • ☐ Contact is added to the client's segment
  • ☐ Slack notification arrives in the client's channel
  • ☐ Thank-you page loads
  • ☐ First drip email sends (check Mautic campaign log)
  • ☐ Matomo records the page view
  • ☐ Meta Pixel fires (check Facebook Events Manager > Test Events)

Phase 3: Tracking & Pixel Setup (15 minutes)

3.1 Meta Pixel

Add to every page in the client's site <head>:

<script>
!function(f,b,e,v,n,t,s)
{if(f.fbq)return;n=f.fbq=function(){n.callMethod?
n.callMethod.apply(n,arguments):n.queue.push(arguments)};
if(!f._fbq)f._fbq=n;n.push=n;n.loaded=!0;n.version='2.0';
n.queue=[];t=b.createElement(e);t.async=!0;
t.src=v;s=b.getElementsByTagName(e)[0];
s.parentNode.insertBefore(t,s)}(window, document,'script',
'https://connect.facebook.net/en_US/fbevents.js');
fbq('init', '251483972902519');
fbq('track', 'PageView');
</script>

On conversion pages add: fbq('track', 'Lead');

On purchase pages add: fbq('track', 'Purchase');

3.2 Matomo Tracking

Add to every page before </body>:

<script>
  var _paq = window._paq = window._paq || [];
  _paq.push(["trackPageView"]);
  _paq.push(["enableLinkTracking"]);
  (function() {
    var u="https://analytics.nunyabunya.com/";
    _paq.push(["setTrackerUrl", u+"matomo.php"]);
    _paq.push(["setSiteId", "CLIENT_SITE_ID"]);
    var d=document, g=d.createElement("script"), s=d.getElementsByTagName("script")[0];
    g.async=true; g.src=u+"matomo.js"; s.parentNode.insertBefore(g,s);
  })();
</script>

3.3 Shlink Tracking Links

Create short links for all marketing URLs:

curl -s -X POST 'https://go.nunyabunya.com/rest/v3/short-urls' \
  -H "X-Api-Key: YOUR_SHLINK_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "longUrl": "https://clientwebsite.com/landing",
    "customSlug": "client-audit",
    "tags": ["client-name", "landing-page"]
  }'

Use go.nunyabunya.com/client-audit in all ads and social posts.

Phase 4: Ad Setup (30 minutes)

4.1 Create Ad Copy

Use the ad copy templates in MASTER PLANS/ad-copy-all-businesses.md as a starting point. Adapt for the client's industry, offer, and voice.

Each client needs minimum:

  • 2 Facebook/Instagram ad variations
  • 2 Google Ads variations (if applicable)

4.2 Create Retargeting Audiences

In Facebook Ads Manager:

  1. Create Custom Audience > Website > All visitors (last 30 days)
  2. Create Custom Audience > Website > Thank-you page visitors (these are converters -- exclude from prospecting)
  3. Create Lookalike Audience from converters (for prospecting)

4.3 Create Campaigns

  • Campaign 1: Prospecting -- target cold audience with lead magnet offer
  • Campaign 2: Retargeting -- target website visitors who didn't convert

Phase 5: Client Handoff (15 minutes)

5.1 Send Client Welcome Email

Use the template in welcomeclientemails agent or write manually:

  • Confirm the services and timeline
  • Share login credentials for any tools they need access to
  • Set expectations for first deliverables
  • Share the Slack channel (if client-facing)
  • Schedule the first check-in call

5.2 Create Metabase Dashboard

For each client, create a dashboard in Metabase showing:

  • New leads this week/month
  • Email open rates
  • Campaign performance
  • Website traffic (if Matomo connected)

Share the dashboard link with the client for transparency.

5.3 Update Internal Tracking

  • Update client_configs.json with all IDs
  • Update the Nunya Bunya Client Roster in GOALS.md
  • Add client to the content calendar (DAILY-CONTENT-SYSTEM.md)
  • Create a project plan in MASTER PLANS/2026/[month]/client-name.md

Automation Opportunity

This entire process should eventually be automated. The goal is an agent where you input:

  • Client name
  • Business type
  • Brand colours
  • Offer details

And it automatically:

  • Creates the Mautic segment, emails, and campaign
  • Generates the landing page HTML
  • Sets up the form handler
  • Creates Shlink tracking links
  • Creates the Slack channel
  • Generates ad copy
  • Outputs everything ready to deploy
