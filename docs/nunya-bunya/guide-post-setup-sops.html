Post-Setup SOPs

Website, Landing Page, Funnel, and Automation Playbook

Purpose: Standard operating procedures for building websites, landing pages, retargeting funnels, and automations after the initial marketing stack is set up. This is how Nunya Bunya delivers for every client, every time.

SOP 1: Building a Website

When to Use

New client needs a website, or existing client needs a redesign.

Tech Stack

  • Static sites: HTML + Tailwind CSS, deployed on Vercel
  • Dynamic sites: Next.js + Tailwind, deployed on Vercel
  • WordPress sites: WordPress on client hosting (for SEO-heavy sites that need a blog CMS)
  • E-commerce: Shopify (client manages, we set up)

Process

Step 1: Discovery (Day 1)

  • Review client's current site (if any)
  • Identify target audience, key services, and competitive positioning
  • Decide on pages needed:
  • Home
  • About
  • Services (one page per service ideally)
  • Contact
  • Blog (if SEO is a priority)
  • Landing pages (separate from main site navigation)

Step 2: Design & Build (Days 2-5)

  • Start from a template or previous client site
  • Apply client branding (colours, fonts, logo)
  • Write or adapt copy for each page
  • Mobile-first design (test on phone before desktop)
  • Every page must include:
  • Clear headline
  • One primary CTA (call, book, buy, download)
  • Social proof (testimonials, logos, reviews)
  • Meta Pixel code in <head>
  • Matomo tracking code before </body>
  • Structured data (LocalBusiness schema for service businesses)
  • Open Graph meta tags for social sharing

Step 3: SEO Setup

  • Title tags: "[Primary Keyword] | [Business Name]" (under 60 chars)
  • Meta descriptions: Include keyword + CTA (under 160 chars)
  • H1: One per page, matches the page's primary keyword
  • Image alt tags: Descriptive, include keywords naturally
  • Internal linking between service pages
  • Google Business Profile optimised and linked
  • Submit sitemap to Google Search Console

Step 4: Deploy

cd /path/to/site
npx vercel --prod

Connect custom domain in Vercel dashboard if not already linked.

Step 5: Verify

  • ☐ All pages load on mobile and desktop
  • ☐ Forms submit correctly
  • ☐ Meta Pixel firing (check Facebook Events Manager > Test Events)
  • ☐ Matomo recording visits (check analytics dashboard)
  • ☐ Page speed acceptable (test at pagespeed.web.dev)
  • ☐ SSL certificate active (https, no warnings)
  • ☐ Google Search Console connected and sitemap submitted

SOP 2: Building a Landing Page

When to Use

Client has a new offer, promotion, lead magnet, or campaign that needs a dedicated conversion page.

Principles

  • One page, one goal, one CTA
  • No navigation menu (don't give them a way to leave)
  • Above the fold: headline + subheadline + CTA button
  • Social proof within scroll
  • Form is short (name + email minimum, phone if high-intent)
  • Fast load time (no heavy images, no unnecessary scripts)

Template Structure

SECTION 1: Hero
  - Headline (what they get)
  - Subheadline (why it matters)
  - CTA button or form
  - Trust indicators (logos, "As seen in", "500+ clients")

SECTION 2: Problem
  - What pain point does this solve?
  - Agitate the problem (make them feel it)

SECTION 3: Solution
  - What you're offering
  - 3-4 bullet points of what's included
  - Image or mockup of the deliverable

SECTION 4: Social Proof
  - Testimonials (with names and photos if possible)
  - Case study snippet
  - Review scores / star ratings

SECTION 5: CTA (repeat)
  - Same form or button as the hero
  - Urgency element if appropriate ("Limited spots", "Offer ends [date]")

SECTION 6: FAQ (optional)
  - 3-5 common objections answered

SECTION 7: Footer
  - Business name, ABN, privacy policy link
  - No navigation links

Building the Page

  1. Duplicate an existing landing page template (e.g., /01-Nunya Bunya/Site/nunyabunya-site/landing.html)
  2. Replace all copy, images, and branding
  3. Update the form action to point to the correct API handler
  4. Add tracking codes:
  • Meta Pixel in <head> (PageView fires automatically)
  • Matomo tracking before </body>
  1. Create the thank-you page (same branding, fires Lead event)
  2. Deploy to Vercel

Form Handler Checklist

Every landing page form must trigger all of the following on submission:

Form submitted
  |
  +--> Create contact in Mautic
  |      - firstname, email, phone
  |      - tag: client-name, campaign-name
  |      - This auto-adds them to the segment
  |      - Segment triggers the drip campaign
  |
  +--> Send Slack notification
  |      - Channel: #client-name
  |      - Message: name, email, phone, interest
  |      - "Follow up within 24 hours"
  |
  +--> Redirect to thank-you page
         - Thank-you page fires fbq('track', 'Lead')
         - Shows confirmation + next steps

Testing

Before going live, submit a test form and verify:

  • ☐ Mautic contact created with correct tags
  • ☐ Contact appears in the right segment
  • ☐ Drip campaign starts (check campaign pending events)
  • ☐ Slack notification received in correct channel
  • ☐ Thank-you page loads
  • ☐ Meta Pixel Lead event fires (Facebook Events Manager > Test Events)
  • ☐ Matomo records the page view and form submission

SOP 3: Setting Up Retargeting

When to Use

Immediately after any landing page or website goes live. Every site should have retargeting from day one.

Creating Retargeting Audiences

In Facebook Ads Manager (business.facebook.com):

Audience 1: All Website Visitors (warm audience)

  • Custom Audiences > Website > All website visitors > Last 30 days
  • Use for: retargeting ads

Audience 2: Converters (exclude from prospecting)

  • Custom Audiences > Website > People who visited specific pages > Thank-you page URL
  • Use for: excluding from ads (they already converted)

Audience 3: Engaged but didn't convert

  • Custom Audiences > Website > All visitors MINUS thank-you page visitors
  • Use for: retargeting with urgency messaging

Audience 4: Lookalike (cold prospecting)

  • Lookalike Audience > Source: Audience 2 (converters) > 1% similarity
  • Use for: finding new people similar to those who already converted

Retargeting Ad Strategy

Ad 1: Reminder (Day 1-7 after visit)

  • "Still thinking about [offer]? Here's why ☑ businesses chose us..."
  • Light touch, value-driven

Ad 2: Social Proof (Day 7-14)

  • Testimonial or case study
  • "See how [client] got [result] in [timeframe]"

Ad 3: Urgency (Day 14-21)

  • "Last chance: [offer] expires [date]"
  • Scarcity or time-limited element

Ad 4: Different Angle (Day 21-30)

  • Different offer or entry point
  • If they didn't want the audit, offer the checklist instead

When to Use

Every client should have at least one complete funnel running. This is the core deliverable.

  |

  |  Social media posts (organic)
  |  Facebook Group posts (free advice with CTA)
  |  LinkedIn outreach
  |  Email newsletter

FORM SUBMISSION (automated)
  |
  +--> Mautic: contact created, tagged, segmented
  +--> Slack: instant notification to team
  +--> Thank-you page: Lead pixel fires, next steps shown
  |
  v
EMAIL DRIP SEQUENCE (automated, via Mautic)
  |
  |  Email 1 (Day 0): Deliver the lead magnet / confirm interest
  |  Email 2 (Day 2-3): Valuable insight or tip
  |  Email 3 (Day 5-7): Case study / social proof
  |  Email 4 (Day 9-10): Address objections
  |  Email 5 (Day 12-14): Direct CTA -- book a call / buy

  |  Make a purchase (Shopify / Stripe)
  |  Request a quote (form)

  |
  +--> Slack: "New booking/purchase from [client]!"
  +--> Mautic: tag updated, moved to "Customer" segment
  +--> CRM: opportunity status updated in Twenty
  +--> Optional: review request email after 7 days

Building the Funnel: Step by Step

1. Define the Offer

  • What's the hook? (Free audit, guide, discount, consultation)
  • What does the prospect get immediately?
  • What's the end goal? (Book a call, buy a product, sign up)

2. Write the Email Sequence

Follow the drip sequence template in drip_sequences.json. For custom sequences:

EmailTimingPurposeTemplate
1ImmediateDeliver the thing, quick win"Here's your [thing]. Start with [action]."
2Day 2-3Educate, build trust"The #1 mistake [audience] makes with [topic]."
3Day 5-7Social proof"How [client] got [result] in [timeframe]."
4Day 9-10Overcome objections"What [thing] actually costs (less than you think)."
5Day 12-14Direct CTA"Ready? [Book/Buy/Start] now."

Each email should be:

  • Under 200 words
  • One clear CTA (not multiple)
  • Personalised with {contactfield=firstname} in Mautic
  • Mobile-friendly (short paragraphs, big buttons)

3. Build the Landing Page (see SOP 2)

4. Set Up Form Handler (see SOP 2)

5. Create Mautic Campaign (see New Client Setup Guide)

6. Create Tracking Links

  • Create a Shlink short link for the landing page
  • Use this link in all ads and social posts
  • Track separately: one link for Facebook, one for LinkedIn, one for organic posts

7. Launch Ads

  • Start with $10-20/day budget
  • Run 2-3 ad variations
  • Prospecting campaign + retargeting campaign
  • Review after 3 days, kill underperformers
  • Scale winners

8. Monitor and Optimise

Daily:

  • Check Slack for new leads
  • Check Mautic for email delivery issues

Weekly:

  • Check Matomo for landing page conversion rate
  • Check email open rates and click rates in Mautic
  • Check ad performance (cost per lead, click-through rate)
  • Check Shlink for link click data

Monthly:

  • Calculate cost per lead and cost per acquisition
  • Update email sequences based on performance
  • Refresh ad creative
  • Report to client

SOP 5: Automation Recipes

Recipe 1: New Lead Notification

Trigger: Form submission on any landing page

Actions:

  1. Create Mautic contact with tags
  2. Post to Slack channel
  3. Create Twenty CRM lead

Implementation: Handled by the form submission API handler (see SOP 2).

Recipe 2: Hot Lead Alert

Trigger: Mautic contact score reaches 50+ points

Actions:

  1. Post urgent Slack alert with fire emoji
  2. Move to "Hot Leads" segment in Mautic
  3. Optional: Create opportunity in Twenty CRM

Implementation: n8n workflow (see n8nwebhooksetup.json, workflow #3).

Mautic point scoring:

ActionPoints
Email opened+5
Email link clicked+10
Landing page visited+10
Form submitted+20
Pricing page visited+15
Multiple visits in 7 days+10

Recipe 3: Follow-Up Reminder

Trigger: Lead submitted form 48 hours ago, no booking made

Actions:

  1. Slack reminder: "Follow up with [name] -- submitted 2 days ago, hasn't booked"

Implementation: Mautic campaign with condition: "Has NOT visited booking-confirmed page within 48 hours" --> Webhook to Slack.

Recipe 4: Review Request

Trigger: Client purchased / completed service 7 days ago

Actions:

  1. Send email: "How was your experience? Leave us a Google review"
  2. Include direct link to Google review page

Implementation: Mautic campaign triggered by "Customer" tag + 7-day delay.

Recipe 5: Re-engagement

Trigger: Contact hasn't opened any email in 30 days

Actions:

  1. Send "We miss you" email with a new offer
  2. If no open in 14 more days, move to "Inactive" segment
  3. Stop sending to avoid spam flags

Implementation: Mautic campaign with inactivity condition.

SOP 6: Reporting

Client Report Template

Monthly report for each client should include:

Summary

  • Total leads generated this month
  • Cost per lead (if running ads)
  • Revenue attributed to marketing (if trackable)

Traffic

  • Total website visitors (Matomo)
  • Top traffic sources
  • Top pages viewed
  • Bounce rate

Email

  • Emails sent
  • Open rate
  • Click rate
  • Unsubscribe rate

Ads (if applicable)

  • Spend
  • Impressions
  • Clicks
  • Cost per click
  • Conversions
  • Cost per conversion

Recommendations

  • What to change next month
  • What's working (do more)
  • What's not working (stop or adjust)

Use Metabase dashboards (https://reports.nunyabunya.com) to generate these automatically where possible.

The Goal: One-Click Client Setup

Eventually, all of this should be a single command:

python3 setup_new_client.py \
  --name "Client Name" \
  --type "photography" \
  --domain "clientwebsite.com" \
  --offer "Free Headshot Guide" \
  --colors "#1a1a1a,#ffffff,#e63946"

This would auto-generate: Mautic segment + emails + campaign, landing page HTML, form handler, Slack channel, Shlink links, ad copy, and Matomo site -- all wired together and ready to deploy.

The agents in NunyaBunyaAgents/ (funnelstrategyagent, offerstrategyagent, etc.) are the foundation for this. Building this meta-agent is a priority.
