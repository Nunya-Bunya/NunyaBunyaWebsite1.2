Biggest Current Problems

  • Core agency truth is spread across multiple folders.
  • Some docs are strategic, some are client-facing, and some are drafts, but that distinction is not obvious from the structure.
  • Offer naming and package logic have evolved faster than the docs were consolidated.
  • Clients, Tools, and Site are large enough that they need stronger conventions.
  • There are review folders and HTML/PDF exports mixed in with source files.

Best Uses For This Workspace

This folder is already strong enough to support:

  • Agency positioning and offer refinement
  • Sales asset production
  • Client onboarding standardization
  • Delivery system design
  • Tool and dashboard development
  • Content engine systemization

The next leverage move is not "make more docs."

It is:

  1. reduce duplication
  2. define canonical ownership
  3. standardize client and system structure
  4. convert the best existing material into an actual operating system

Next Recommended Cleanup Sequence

  1. Finalize the target folder structure in RESTRUCTURE-PLAN.md
  2. Consolidate overlapping strategy docs into one internal master set
  3. Separate source docs from exports and review copies
  4. Standardize the Clients folder template
  5. Audit Tools into active, incubating, and archive buckets

Working Rule

Until the migration is complete:

  • Prefer the markdown source over HTML/PDF exports
  • Prefer root-level strategic docs over duplicated review versions
  • Prefer docs referenced here over similarly named files elsewhere
