The $16/Month Marketing Stack

A Complete Guide to Setting Up Professional Marketing Infrastructure with Open-Source Software

By Nunya Bunya | nunyabunya.com

What This Guide Is

This is a step-by-step guide to building a professional marketing system for your small business using free and open-source software. By the end, you will have:

  • A CRM to manage all your leads and clients
  • Email automation that sends the right message at the right time
  • Website analytics that tell you exactly what's working
  • A link shortener that tracks every click
  • A booking system for consultations and appointments
  • Landing pages that capture leads 24/7
  • Dashboards that show your marketing performance at a glance

Total cost: approximately $16 AUD/month for hosting. Everything else is free.

What You Need Before Starting

  1. A domain name -- e.g., yourbusiness.com.au (purchase from Namecheap, ~$15/year)
  2. A VPS (Virtual Private Server) -- this is where your tools live. We recommend Hetzner Cloud CX22 (~$16 AUD/month for 4 vCPU, 8GB RAM, 80GB storage)
  3. Basic comfort with a terminal/command line -- you will need to copy and paste commands. No coding required.
  4. About 2-3 hours for initial setup

Part 1: Get Your Server

Step 1: Sign Up for Hetzner Cloud

  1. Go to https://www.hetzner.com/cloud
  2. Create an account
  3. Create a new project (e.g., "Marketing Stack")
  4. Click "Add Server"
  5. Choose:
  • Location: Australia (or closest to your customers)
  • Image: Ubuntu 24.04
  • Type: CX22 (4 vCPU, 8GB RAM) -- this runs everything comfortably
  • SSH Key: Create one (Hetzner will walk you through this)
  1. Click "Create & Buy Now"
  2. Note your server's IP address (e.g., 5.223.79.5)

Step 2: Point Your Subdomains to the Server

Go to your domain registrar (Namecheap, GoDaddy, etc.) and add these DNS records. All should be A records pointing to your server's IP address:

TypeNameValue
Amail.yourbusiness.comYOURSERVERIP
Acrm.yourbusiness.comYOURSERVERIP
Aanalytics.yourbusiness.comYOURSERVERIP
Areports.yourbusiness.comYOURSERVERIP
Ago.yourbusiness.comYOURSERVERIP
Alinks.yourbusiness.comYOURSERVERIP

Replace yourbusiness.com with your actual domain and YOURSERVERIP with the IP from Step 1.

Step 3: Set Up the Server

Open your terminal (Mac: Terminal app, Windows: PowerShell) and connect:

ssh root@YOUR_SERVER_IP

Then run these commands to install Docker (the tool that runs all your marketing software):

# Update the system
apt-get update && apt-get upgrade -y

# Install Docker
curl -fsSL https://get.docker.com | sh
systemctl enable docker && systemctl start docker

# Install useful tools
apt-get install -y curl wget htop ufw fail2ban

# Set up firewall (only allow web traffic and SSH)
ufw default deny incoming
ufw default allow outgoing
ufw allow 22/tcp
ufw allow 80/tcp
ufw allow 443/tcp
ufw --force enable

# Create the directory for your marketing stack
mkdir -p /opt/marketing-stack

Part 2: Install the Marketing Stack

Step 4: Create Your Configuration

Still on your server, create the environment file that stores all your settings:

cd /opt/marketing-stack
nano .env

Paste in the following (replace ALL values in CAPS with your own):

# General
DOMAIN=yourbusiness.com
TIMEZONE=Australia/Brisbane

# Traefik (SSL/proxy)
ACME_EMAIL=you@yourbusiness.com

# MySQL
MYSQL_ROOT_PASSWORD=GENERATE_A_LONG_RANDOM_PASSWORD_HERE

# Mautic (email automation)
MAUTIC_DB_USER=mautic
MAUTIC_DB_PASSWORD=GENERATE_ANOTHER_PASSWORD
MAUTIC_FROM_NAME=Your Business Name
MAUTIC_FROM_EMAIL=hello@yourbusiness.com

# Matomo (analytics)
MATOMO_DB_USER=matomo
MATOMO_DB_PASSWORD=GENERATE_ANOTHER_PASSWORD

# Shlink (link shortener)
SHLINK_DB_USER=shlink
SHLINK_DB_PASSWORD=GENERATE_ANOTHER_PASSWORD
SHLINK_API_KEY=GENERATE_A_64_CHAR_HEX_STRING

# SMTP (email sending -- use Amazon SES)
SMTP_HOST=email-smtp.ap-southeast-2.amazonaws.com
SMTP_PORT=587
SMTP_USER=YOUR_SES_SMTP_USERNAME
SMTP_PASSWORD=YOUR_SES_SMTP_PASSWORD

To generate random passwords, run this command for each one:

openssl rand -hex 24

Save and exit (Ctrl+X, then Y, then Enter).

Step 5: Create the Docker Compose File

nano docker-compose.yml

This is the file that defines all your services. It's long, but you only set it up once:

      - "--entrypoints.websecure.address=:443"
      - "--certificatesresolvers.letsencrypt.acme.httpchallenge.entrypoint=web"
      - "--certificatesresolvers.letsencrypt.acme.email=${ACME_EMAIL}"
      - "--certificatesresolvers.letsencrypt.acme.storage=/letsencrypt/acme.json"

  mysql:
    image: mysql:8.0
    container_name: nb-mysql
    restart: unless-stopped
    environment:
      MYSQL_ROOT_PASSWORD: ${MYSQL_ROOT_PASSWORD}

Step 6: Create the MySQL Databases

mkdir -p mysql/init
nano mysql/init/01-create-databases.sh

Paste:

#!/bin/bash
mysql -u root -p"${MYSQL_ROOT_PASSWORD}" <<-EOSQL
    CREATE DATABASE IF NOT EXISTS mautic_db;
    CREATE DATABASE IF NOT EXISTS matomo_db;
    CREATE DATABASE IF NOT EXISTS shlink_db;
    GRANT ALL ON mautic_db.* TO '${MAUTIC_DB_USER}'@'%' IDENTIFIED BY '${MAUTIC_DB_PASSWORD}';
    GRANT ALL ON matomo_db.* TO '${MATOMO_DB_USER}'@'%' IDENTIFIED BY '${MATOMO_DB_PASSWORD}';
    GRANT ALL ON shlink_db.* TO '${SHLINK_DB_USER}'@'%' IDENTIFIED BY '${SHLINK_DB_PASSWORD}';
    FLUSH PRIVILEGES;
EOSQL

Save and exit. Make it executable:

chmod +x mysql/init/01-create-databases.sh

Step 7: Launch Everything

docker compose up -d

Wait about 60 seconds, then check all containers are running:

docker compose ps

You should see all containers with status "Up."

Part 3: Set Up Each Tool

Mautic (Email Automation) -- https://mail.yourbusiness.com

  1. Open the URL in your browser
  2. Complete the setup wizard:
  • Create your admin account (save these credentials!)
  • Database settings should auto-populate
  1. After setup, go to Settings > Configuration > API Settings:
  • Enable API: Yes
  • Enable HTTP Basic Auth: Yes
  1. Go to Settings > Configuration > Email Settings:
  • Mailer is already configured from the Docker environment
  • Send a test email to verify it works

What Mautic does for you:

  • Stores all your leads/contacts
  • Sends automated email sequences (drip campaigns)
  • Tracks email opens and clicks
  • Segments contacts by interest, behaviour, or source
  • Scores leads so you know who's hot

Matomo (Analytics) -- https://analytics.yourbusiness.com

  1. Open the URL and complete the setup wizard
  2. Create your admin account
  3. Add your website as a site to track
  4. Matomo will give you a JavaScript tracking code -- add this to every page on your website, just before </body>:
<script>
  var _paq = window._paq = window._paq || [];
  _paq.push(["trackPageView"]);
  _paq.push(["enableLinkTracking"]);
  (function() {
    var u="https://analytics.yourbusiness.com/";
    _paq.push(["setTrackerUrl", u+"matomo.php"]);
    _paq.push(["setSiteId", "1"]);
    var d=document, g=d.createElement("script"), s=d.getElementsByTagName("script")[0];
    g.async=true; g.src=u+"matomo.js"; s.parentNode.insertBefore(g,s);
  })();
</script>

What Matomo does for you:

  • Shows who visits your website
  • Which pages they view
  • Where they come from (Google, Facebook, direct)
  • How long they stay
  • What they click on
  • All without sending data to Google

Metabase (Dashboards) -- https://reports.yourbusiness.com

  1. Open the URL and create your admin account
  2. Connect your data sources:
  • Add MySQL connection (host: mysql, port: 3306, database: mautic_db)
  • This lets you build dashboards from your Mautic email data
  1. Create your first dashboard:
  • New leads this week
  • Email open rates
  • Top performing campaigns

What Metabase does for you:

  • Beautiful dashboards showing your marketing performance
  • No SQL knowledge needed -- point-and-click interface
  • Schedule reports to email you weekly
  • Share dashboards with your team

Shlink (Link Shortener) -- https://links.yourbusiness.com

  1. Open the URL
  2. Connect it to your Shlink API:
  • Server URL: https://go.yourbusiness.com
  • API Key: the one you set in .env
  1. Create your first short link

What Shlink does for you:

  • Shortens URLs (go.yourbusiness.com/audit instead of a long URL)
  • Tracks every click (when, where, what device)
  • Shows which marketing channels drive the most traffic
  • Lets you update the destination URL without changing the link

Cal.com (Booking) -- https://cal.com

  1. Sign up for a free account at cal.com
  2. Set your availability
  3. Create booking types (e.g., "Free 30-Minute Strategy Call")
  4. Use the booking link in your emails and landing pages

What Cal.com does for you:

  • Lets people book time with you without back-and-forth emails
  • Syncs with your Google Calendar
  • Sends automatic reminders
  • Free for individual use

Part 4: Set Up Your First Marketing Funnel

Now that the tools are running, here's how to use them together.

The Funnel Flow

Person sees your ad or social post
  --> Clicks link (tracked by Shlink)
  --> Lands on your landing page (tracked by Matomo)
  --> Fills out form (lead captured in Mautic)
  --> Receives automated email sequence (sent by Mautic)
  --> Books a call (via Cal.com)
  --> You close the deal

Step 1: Create a Lead Magnet

A lead magnet is something valuable you give away for free in exchange for an email address. Examples:

  • A checklist relevant to your industry
  • A guide or how-to PDF
  • A free audit or assessment
  • A discount code

Step 2: Create a Landing Page

A landing page is a single-purpose page designed to capture leads. It should have:

  • A clear headline
  • What they get (the lead magnet)
  • A simple form (name + email minimum)
  • A call to action button

You can build this with any website builder, or create a simple HTML page.

Step 3: Set Up Mautic Email Sequence

In Mautic:

  1. Create a segment for this campaign (e.g., "Free Audit Leads")
  2. Create your emails (typically 3-5 emails over 7-14 days):
  • Email 1 (Day 0): Deliver the lead magnet + a quick win
  • Email 2 (Day 2): Share a helpful tip or insight
  • Email 3 (Day 5): Show a case study or result
  • Email 4 (Day 9): Address common objections
  • Email 5 (Day 14): Direct call to action (book a call)
  1. Create a campaign that wires it all together:
  • Trigger: Contact added to segment
  • Send Email 1 immediately
  • Wait 2 days, send Email 2
  • Wait 3 days, send Email 3
  • etc.
  1. Publish the campaign

Step 4: Connect Your Form to Mautic

Your landing page form needs to send data to Mautic. The simplest way:

Use Mautic's built-in form builder (under Components > Forms), then embed it on your landing page using the provided HTML snippet.

Or, if you have a custom form, POST the data to Mautic's API:

fetch('https://mail.yourbusiness.com/api/contacts/new', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
    'Authorization': 'Basic ' + btoa('username:password')
  },
  body: JSON.stringify({
    firstname: formData.name,
    email: formData.email,
    tags: ['lead-magnet', 'your-campaign-name']
  })
});

Step 5: Track Everything

  • Add the Matomo tracking code to your landing page
  • Create a Shlink short link for your landing page URL
  • Use the short link in your ads and social posts
  • Check Matomo for page views and Mautic for conversions

Part 5: Set Up Email Sending (Amazon SES)

Mautic needs an email delivery service. Amazon SES is the cheapest at scale.

  1. Sign up at aws.amazon.com
  2. Go to Amazon SES in the console
  3. Switch to your nearest region (e.g., ap-southeast-2 for Australia)
  4. Verify your domain:
  • Create identity > Domain > yourbusiness.com
  • Add the DKIM CNAME records to your DNS
  • Add the DMARC TXT record
  1. Request production access (fill in the form explaining you send opt-in transactional emails)
  2. Create SMTP credentials (SES > SMTP settings > Create SMTP credentials)
  3. Put the credentials in your .env file

Cost: under $1/month for most small businesses (first 62,000 emails/month are essentially free if sending from EC2).

To show ads to people who visited your website but didn't convert:

  1. Go to business.facebook.com/events_manager

  3. Add the pixel code to every page on your website, in the <head>:

Part 7: Ongoing Maintenance

Daily (5 minutes)

  • Check Mautic for new leads
  • Respond to any booking requests

Weekly (30 minutes)

  • Review Matomo dashboard (traffic, top pages, sources)
  • Review Mautic campaign performance (open rates, click rates)
  • Review Shlink click data (which links are getting traffic)

Monthly (1 hour)

  • Review Metabase dashboards
  • Clean up Mautic contacts (remove bounces, unsubscribes)
  • Update email sequences based on performance
  • Check Mautic cron container is running (docker compose ps)

Quarterly

  • Update your lead magnet if needed
  • Review and refresh ad creative
  • Check for software updates (docker compose pull && docker compose up -d)
