Nunya Bunya -- Client Folder Template

Purpose: Standardize how every client project is organized.

Use for: New Bedrock clients, retainer clients, internal brands, and case-study projects.

Standard Structure

Client-Name/
├── 00-admin/
├── 01-strategy/
├── 02-brand/
├── 03-site/
├── 04-campaigns/
├── 05-content/
├── 06-reporting/
└── 99-archive/

Folder Rules

00-admin

Use for operational details:

  • client summary
  • contact details
  • contract and billing notes
  • access checklist
  • kickoff notes
  • project status

Do not store passwords directly in this folder.

Use it to document where access is stored and who owns it.

01-strategy

Use for business and marketing strategy:

  • intake survey
  • research
  • ideal customer profile
  • offer map
  • messaging framework
  • competitor analysis
  • funnel map

02-brand

Use for identity and creative direction:

  • brand DNA
  • tone of voice
  • style guide
  • logo files or references
  • color and typography notes
  • visual examples
  • brand bible

03-site

Use for website and landing page work:

  • website source files
  • landing pages
  • copy docs
  • sitemap
  • tracking notes
  • deployment links
  • preview links

04-campaigns

Use for conversion campaigns:

  • ad copy
  • targeting notes
  • lead magnets
  • email sequences
  • n8n/Mautic workflow notes
  • booking flow
  • campaign results

05-content

Use for ongoing content:

  • content calendar
  • post bank
  • social captions
  • blog drafts
  • video scripts
  • creative briefs
  • approved/scheduled content notes

06-reporting

Use for performance and proof:

  • weekly reports
  • monthly reports
  • dashboard links
  • before/after screenshots
  • metrics
  • case study notes
  • testimonials

99-archive

Use for retired, superseded, or old work:

  • previous drafts
  • old exports
  • replaced assets
  • abandoned directions

Do not delete useful client history.

Archive it.

Naming Convention

Use lowercase kebab-case for folders:

brisbane-zinia-massage
pawesome-dog-walkers
conner-injury-law

Avoid:

  • spaces
  • inconsistent capitalization
  • duplicate folders for the same client

Minimum Files For A New Client

Every client should start with:

  • 00-admin/CLIENT-SUMMARY.md
  • 00-admin/STATUS.md
  • 00-admin/ACCESS-CHECKLIST.md
  • 01-strategy/INTAKE.md
  • 01-strategy/OFFER-MAP.md
  • 04-campaigns/CAMPAIGN-PLAN.md
  • 06-reporting/CASE-STUDY-NOTES.md

Lifecycle Status

Use one of these statuses in 00-admin/STATUS.md:

  • prospect
  • active-bedrock
  • active-retainer
  • paused
  • complete
  • archived

New Client Setup Steps

  1. Copy Clients/_TEMPLATE
  2. Rename it to the client slug
  3. Fill in 00-admin/CLIENT-SUMMARY.md
  4. Fill in 00-admin/STATUS.md
  5. Add intake details to 01-strategy/INTAKE.md
  6. Add core offer/funnel to 01-strategy/OFFER-MAP.md
  7. Create the first campaign plan in 04-campaigns/CAMPAIGN-PLAN.md
  8. Start collecting proof in 06-reporting/CASE-STUDY-NOTES.md

Existing Client Cleanup Rule

Do not reorganize existing client folders all at once.

For each existing client:

  1. create the standard folders
  2. move obvious files only
  3. leave unclear files in place until reviewed
  4. create a MIGRATION-NOTES.md file documenting what changed
