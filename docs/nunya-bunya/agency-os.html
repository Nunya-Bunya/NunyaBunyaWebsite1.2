Nunya Bunya -- Agency OS

Purpose: A condensed internal operating document for Nunya Bunya.

This is the fast, practical version of the agency pulled from the current source-of-truth, business plan, offer docs, and action docs.

Status: Working consolidation draft created on 2026-04-13.

1. What Nunya Bunya Is

Nunya Bunya is an AI-powered digital marketing agency serving Brisbane and Southeast Queensland small businesses.

The real differentiator is not "AI."

It is:

  • full-stack delivery
  • self-hosted infrastructure
  • lower structural operating cost
  • human-reviewed output
  • documented systems instead of agency mystery

Core idea:

We do not just run marketing.
We build a machine.

2. Current Reality

  • Main business: Nunya Bunya
  • Stage: pre-revenue from agency clients
  • Paying external clients: 0
  • Internal clients exist and act as proving grounds
  • Core infrastructure appears largely built
  • The main bottleneck is no longer tooling
  • The main bottleneck is first paying clients, proof, packaging, and controlled execution

This matters because the agency does not currently need more abstract planning.

It needs:

  • one clean operating model
  • one clear offer architecture
  • one proof-of-results story
  • one repeatable client delivery flow

3. Positioning

Audience

Small-to-medium businesses in Brisbane and Southeast Queensland, especially:

  • trades
  • health and wellness
  • hospitality
  • professional services
  • other local service businesses

Positioning Statement

Nunya Bunya is a Brisbane AI-powered agency that builds the actual marketing system for small businesses: website, landing pages, ads, email, reporting, automation, and content, with human review before anything goes live.

Why It Wins

  • lower tech overhead than traditional agencies
  • broader system ownership than piecemeal freelancers
  • more transparency than locked-in agencies
  • more operational leverage than manual shops

4. Core Operating Principles

These appear to be the right principles to keep enforcing:

  • If a human will see it, Ben sees it first.
  • The workbench is the product.
  • The approval queue is not overhead. It is the quality filter.
  • AI drafts. Humans make it premium.
  • Client ownership and portability are selling points.
  • Tool cost discipline is part of the business model.

5. Productized Offers

Current Offer Families

Bedrock

One-time build offers that create the marketing foundation.

  • Bedrock Lite
  • Bedrock
  • Bedrock Pro

These are currently the clearest entry point because they:

  • solve an obvious setup problem
  • are easier to sell than abstract retainers
  • create proof assets fast
  • can convert into monthly retainers

Retainers

  • NB Onyx
  • NB Tanzanite
  • NB Black Opal

These appear strategically right, but the lower and middle tiers are not yet defined tightly enough in the docs.

That means Bedrock is likely the better near-term sales anchor until retainer scope is sharper.

6. Immediate Business Priorities

The real order of operations should be:

  1. Land the first paying client
  2. Use the first client to prove the system and produce a case study
  3. Tighten the Bedrock offer and fulfillment workflow
  4. Standardize review and approval
  5. Sharpen retainer definitions
  6. Scale only after the workflow is stable

Right now, the business risk is not underbuilding.

It is overbuilding without revenue proof.

7. Delivery Model

Nunya Bunya should operate like this:

  • Strategy and review stay centralized
  • Production is systemized
  • Contractors can support execution later
  • No client-facing output ships unreviewed
  • Internal systems should reduce setup and reporting friction first

This means the most important internal systems are:

  • onboarding orchestration
  • approval queue
  • client dashboard/reporting
  • package fulfillment templates

8. Best Near-Term Sales Logic

The best path suggested by the current docs is:

  1. Sell or trade one Bedrock project
  2. Overdocument the before and after
  3. Turn that into a case study
  4. Use that proof to sell more Bedrock projects
  5. Convert strong-fit clients into retainers

This is cleaner than trying to force high-ticket retainers before proof exists.

9. Biggest Gaps

These are the main gaps I see from the current docs:

  • offer definitions are still uneven
  • core truth is spread across too many files
  • there is not yet one obvious client folder standard
  • there is not yet one obvious agency dashboard or approval workbench
  • tool sprawl may be outrunning operating discipline

None of that is fatal.

It just means the next growth phase should be consolidation, not more fragmentation.

10. Recommended Canonical Doc Set

Nunya Bunya should eventually have a small internal master set:

  1. Agency OS
  2. Source of Truth
  3. Master Actions
  4. Offer Architecture
  5. SOP Master Index
  6. Client System Template
  7. Tooling Inventory

Everything else should either support one of those or become archive material.

Current offer architecture now lives in OFFER-ARCHITECTURE.md.

11. What To Build Next Internally

If effort is limited, the highest-leverage internal moves are:

  1. Consolidate strategy and offer docs
  2. Standardize client structure
  3. Audit Tools into active vs incubating vs archive
  4. Build an approval/review workbench
  5. Productize Bedrock fulfillment into a repeatable system

12. Working Conclusion

Nunya Bunya already has enough depth to become a serious agency system.

The next win is not more ideas.

It is compression:

  • fewer canonical docs
  • clearer structure
  • sharper offers
  • standardized client handling
  • better review flow

That is how this folder turns from "a lot of good work" into "an actual machine."
