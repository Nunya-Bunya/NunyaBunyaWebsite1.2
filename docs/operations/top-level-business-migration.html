Top-Level Business Migration

Purpose: Move Ben-owned businesses out of 03-Nunya Bunya/Clients and into their own top-level folders without breaking active websites, scripts, or deploy paths.

New Canonical Rule

Every business owns its own files.

03-Nunya Bunya/Clients should be for external clients and Nunya Bunya delivery work. It should not be the long-term home for Ben-owned ventures.

Safe Migration Steps

For each business:

  • Inventory both the nested folder and the top-level folder.
  • Identify active site/deploy paths.
  • Move files into the matching top-level structure.
  • Leave a short MIGRATED.md note in the old nested folder.
  • Update COMPANY-CONTROL.md.
  • Update any master indexes.
  • Run or inspect deploy commands before deleting or archiving the old path.

Do Not Move Yet Without Checking

  • Active website source folders.
  • Folders with Vercel configuration.
  • Scripts that use hardcoded paths.
  • Anything tied to a live domain, booking flow, form endpoint, or deployed API.

Immediate Recommendation

Start with Power Portraits because it is Priority 1 and the target root folder is almost empty.
