1. THE SYSTEM AT A GLANCE

CLIENT SIGNS → Payment clears
  → Nextcloud folder auto-created (n8n webhook)
  → Twenty CRM deal → "Active"
  → Mautic tag → "client"
  → Opening Gambit begins

OPENING GAMBIT (Audit)
  → Marketing audit + competitor analysis
  → Strategy pack generated
  → Deliverables → Nextcloud /01-Audit/ + /02-Strategy/
  → [HUMAN] Ben reviews + presents to client

THE FITOUT (Build)
  → Website, brand, content, email sequences, booking, tracking
  → All assets → Nextcloud /03-Brand/ through /07-Website/
  → [HUMAN] Ben reviews each deliverable before client sees it

THE POLISH (Quality Gate)
  → Re-audit everything built
  → Fix issues, record KPI baselines
  → Final handoff package → Nextcloud /08-Delivery/
  → [HUMAN] Ben presents final system to client

ONGOING RETAINER ($3,500/mo)
  → Monthly content packs
  → Monthly analytics reports
  → Email campaigns
  → Ad management
  → Website updates
  → Everything tracked in Matomo + Metabase

3. THE CLIENT JOURNEY (Client's POV)

What the client sees:

1. DISCOVERY
   → Finds NB via content, referral, or outreach
   → Visits nunyabunya.com
   → Downloads lead magnet OR books a call via Cal.com

2. SALES
   → Gets 5-email nurture sequence (Mautic, automatic)
   → Books discovery call (Cal.com)
   → [HUMAN] Ben runs the call
   → Receives proposal
   → Signs + pays

3. ONBOARDING
   → Gets welcome email
   → Receives link to their Nextcloud portal (files.nunyabunya.com)
   → Fills out client intake questionnaire
   → [HUMAN] Ben explains the 3-phase process

4. OPENING GAMBIT (Week 1-2)
   → Client sees: audit report + strategy pack appear in their portal
   → [HUMAN] Ben presents findings in a call

5. THE FITOUT (Week 3-6)
   → Client sees: assets appearing in their portal folders
   → Brand assets, website preview, content samples
   → [HUMAN] Ben gets approval at key checkpoints
   → Website goes live
   → Booking system goes live
   → Email sequences activated

6. THE POLISH (Week 7-8)
   → Client sees: final delivery package in /08-Delivery/
   → Everything tested and verified
   → [HUMAN] Ben walks client through their full system

7. ONGOING
   → Client sees: monthly content in their portal
   → Client sees: monthly analytics report in /06-Reports/
   → Client sees: results improving over time
   → [HUMAN] Monthly check-in call

What the client NEVER sees:

  • n8n workflows
  • Hetzner server
  • Slack channels
  • MASTER PLANS
  • Cron jobs
  • Internal tooling
  • How assets are generated (AI or manual)

4. THE OPERATOR JOURNEY (Ben's POV)

Daily routine (pulled from MASTER-ACTIONS.md):

MORNING (automated before Ben wakes up):
  05:55 AEST → Daily taskmaster generates today's tasks (cron)
  06:00 AEST → Pipeline runner executes scheduled SOPs (cron)
  07:00 AEST → Daily digest hits #daily-digest in Slack (cron)
  07:00 AEST → Daily briefing assembled (cron)
  08:00 AEST → ORCA Pulse digest (cron)

  → Check #alerts and #outreach in Slack
  → Check #daily-digest for overnight activity
  → Respond to DMs, emails, inbound leads
  → Update Twenty CRM with new leads
  → Write and post content (NB + BAC)

  → Update MASTER-ACTIONS.md with progress

11. HOW DELIVERABLES GET DELIVERED

The flow:

Ben creates asset (local Mac tools)
  → Asset saved to local project folder
  → Asset uploaded to Nextcloud client folder
  → Ben reviews in Nextcloud (visual check)
  → Client notified (email or Slack or call)
  → Client views in their Nextcloud portal

Notification methods:

TypeChannelWhen
Major milestone (audit, website launch)Video call + emailEnd of each phase
Monthly content packEmail with portal link1st week of month
Monthly reportEmail with portal linkLast week of month
Quick updateEmail or textAs needed
Urgent (site down, campaign issue)Phone callImmediately

Upload automation (future):

  • n8n workflow: watch local folder → auto-upload to Nextcloud
  • n8n workflow: Nextcloud upload → notify client via email
  • Currently: manual upload via Nextcloud web UI or WebDAV API
