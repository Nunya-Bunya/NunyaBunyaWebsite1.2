1. Twitter Automation

Tweet Queue Poster

  • Source: n8n
  • Status: Active
  • Schedule: 0 9,12,15,18 1-5 — 9am, 12pm, 3pm, 6pm AEST (weekdays)
  • What it does: Fetches next approved tweet from Supabase tweet_queue, posts to Twitter, marks as posted, notifies Slack
  • Tracking: Tweets posted count, post success/fail rate, Twitter post IDs
  • Results: Twitter (posted) | Supabase tweet_queue | Slack #announcements
  • Workflow file: 00-Ben HQ/marketing-stack/scripts/n8nworkflowtweet_poster.json

Blog to Tweets Generator

  • Source: n8n
  • Status: Active
  • Schedule: 0 /2 — Every 2 hours
  • What it does: Fetches unprocessed approved blog posts from contentcampaigns, generates 5 brand-specific tweets via Claude API, stores as drafts in tweetqueue
  • Tracking: Drafts generated per blog post, blog posts processed, queue depth
  • Results: Supabase tweet_queue (drafts) | Slack #announcements
  • Workflow file: 00-Ben HQ/marketing-stack/scripts/n8nworkflowblogtotweets.json
  • Brands: Nunya Bunya, ORCA Film Awards, Power Portraits, Ben Alek Conner

Tweet Engagement (Repost/Reply)

  • Source: n8n
  • Status: Active
  • Schedule: 0 10,14 1-5 — 10am & 2pm AEST (weekdays)
  • What it does: 10am: replies with thread to recent tweets. 2pm: reposts. Keeps engagement active on posted content
  • Tracking: Replies sent, reposts made, engagement rate change
  • Results: Twitter (engagements) | Supabase tweet_queue | Slack #announcements
  • Workflow file: 00-Ben HQ/marketing-stack/scripts/n8nworkflowtweet_engagement.json

Tweet Metrics Tracker

  • Source: n8n
  • Status: Active
  • Schedule: 0 21 * — Daily 7am AEST
  • What it does: Fetches impressions, engagement, and likes for all posted tweets. Builds daily summary
  • Tracking: Impressions per tweet, engagement rate, likes & retweets, top performer
  • Results: Supabase (metrics stored) | Slack #announcements (daily summary)
  • Workflow file: 00-Ben HQ/marketing-stack/scripts/n8nworkflowtweet_metrics.json

2. Content Distribution

Organic Engine

  • Source: n8n
  • Status: Active
  • Schedule: 0 8,11,14,17 1-5 — 8am, 11am, 2pm, 5pm AEST (weekdays)
  • What it does: Fetches approved/scheduled posts from outreach_posts table. Distributes to approved channels. Marks as posted
  • Tracking: Posts distributed, platform breakdown, post success rate
  • Results: Supabase outreach_posts | Cross-platform posts
  • Workflow file: 00-Ben HQ/marketing-stack/scripts/n8nworkfloworganic_engine.json

ORCA Auto-Publish Article

  • Source: n8n
  • Status: Active (webhook trigger)
  • Schedule: On new article creation (webhook)
  • What it does: Receives article metadata, generates platform-specific copy (Twitter, Email, Slack), creates tracked Shlink URLs, posts to Twitter, queues email in Mautic
  • Tracking: Articles published, Shlink clicks, emails queued
  • Results: Twitter (tweet) | Mautic (email queued) | Slack (notification)
  • Workflow file: 00-Ben HQ/marketing-stack/scripts/n8nworkfloworcaarticlepublish.json

3. Outreach

Daily Twitter Outreach

  • Source: Python (Hetzner VPS)
  • Status: Active
  • Schedule: 0 22 0-4 — Daily 8am AEST (Mon-Fri)
  • What it does: Searches Twitter for Brisbane professionals (50+ followers). Auto-follows 5 prospects, auto-DMs top 1, posts other 4 to Slack with pre-written DM templates
  • Tracking: Prospects found, auto-follows sent, DMs sent, Slack templates posted
  • Results: Slack #outreach | Supabase outreach_log | Twitter (follows & DMs)
  • Script: /opt/tweet-poster/daily-outreach.py on Hetzner
  • Known issue: Twitter search API returns non-Brisbane users. Backburner: improve via Apify

Auto Email Outreach

  • Source: Python
  • Status: Active
  • Schedule: /30 * — Every 30 minutes
  • What it does: Picks next untouched contact from Mautic prospect segment, generates personalized cold email via Claude API, sends through Mautic
  • Tracking: Emails sent, contacts remaining in queue, send rate
  • Results: Mautic (email sent via API)
  • Script: 00-Ben HQ/scripts/auto-email-outreach.py
  • State file: .email-outreach-state.json

Prospect Enrichment (Apollo)

  • Source: Vercel Edge Function
  • Status: Active
  • Schedule: Daily (3 prospects, 10 on last day of month)
  • What it does: Processes prospect list from Supabase, enriches missing contact info (name, email, phone) via Apollo API
  • Tracking: Prospects enriched, fields filled (email, phone, name), API credits used
  • Results: Supabase prospects table (enriched)
  • Script: 00-Ben HQ/Site/copy-of-nunya-bunya-hq/api/cron-enrich.ts

4. Daily Operations

Daily Task Dashboard

  • Source: Python
  • Status: Active
  • Schedule: 0 21 * — Daily 7am AEST
  • What it does: Posts morning task dashboard to each business Slack channel with today's tasks. Color-coded urgency (white=today, yellow=1d overdue, orange=2d, red=3d+)
  • Tracking: Tasks posted per channel, overdue tasks count, completion rate
  • Results: 7 business Slack channels + #alerts

  • Tracking: Emails sent, campaigns triggered, segments updated, webhooks processed
  • Results: Mautic MySQL database | Internal queues
  • Config: 00-Ben HQ/marketing-stack/docker-compose.yml

Voice Memo Pipeline

  • Source: n8n (webhook trigger)
  • Status: Active
  • Schedule: On voice memo upload (webhook)

  • Workflow file: 00-Ben HQ/marketing-stack/scripts/n8nvoicememo_workflow.json
