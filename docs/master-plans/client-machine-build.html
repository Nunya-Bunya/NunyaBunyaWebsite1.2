Client Machine Build Plan

Get Nunya Bunya to "ready to take clients" as fast as possible

Created: 2026-03-28

Goal: Complete the infrastructure so that onboarding a new client takes under 2 hours, and monthly management takes under 5 hours per client.

Current state: Stack is deployed, Mautic provisioned, emails created, campaigns built, pixel live on 4 sites. But the client-facing pipeline isn't complete -- no way to actually sign, onboard, and deliver for a paying client end-to-end.

What "Ready to Take Clients" Means

A client signs. Within 48 hours they have:

  • ☐ A live landing page with their branding, offer, and form
  • ☐ Form connected to Mautic (auto-creates contact, tags, starts drip)
  • ☐ Slack notifications on every lead
  • ☐ 3-5 email drip sequence running automatically
  • ☐ Meta Pixel retargeting on all pages
  • ☐ Matomo analytics tracking
  • ☐ Shlink tracking links for all campaigns
  • ☐ Ad campaigns live (Meta prospecting + retargeting)
  • ☐ Metabase dashboard showing their leads and performance
  • ☐ Cal.com booking link for consultations

And every month, mostly on autopilot:

  • ☐ Ads running and optimised
  • ☐ Drip emails sending
  • ☐ Leads flowing to Slack
  • ☐ Monthly report auto-generated
  • ☐ One client call

Build Phases

PHASE 1: Fix the Gaps (Day 1)

Time: 3-4 hours

What: Finish the things that are 90% done but not working end-to-end.

  • ☐ 1.1 Verify SES production access is approved
  • Check AWS SES console
  • If not approved, follow up or send test to verified email only
  • Send a test email from Mautic to confirm delivery works
  • ☐ 1.2 Publish Mautic campaigns
  • Log into https://mautic.nunyabunya.com
  • Review all 6 campaigns (BAC, Nunya Bunya, Conner Law, ORCA, Power Portraits, Bella Rhyder)
  • Check email content looks right
  • Publish each campaign
  • ☐ 1.3 Set up Cal.com
  • Create account at cal.com (or log in if existing)
  • Create booking type: "Free 30-Minute Strategy Call"
  • Set availability (your actual working hours)
  • Get the booking URL
  • Update all landing page CTAs and email CTAs with the real booking link
  • ☐ 1.4 Test the full pipeline end-to-end
  • Submit a form on nunyabunya.com/free-audit
  • Verify: Mautic contact created, Slack notification received, drip email #1 arrives
  • Submit a form on powerportraitsphotography.com/lead-magnet
  • Verify same pipeline works
  • Fix anything broken

PHASE 2: Client Onboarding Template (Day 2)

Time: 3-4 hours

What: Build reusable templates so every new client setup is copy-paste, not start-from-scratch.

  • ☐ 2.1 Landing page template
  • Create a clean, brandable HTML template at /01-Nunya Bunya/Templates/landing-page-template/
  • Includes: hero section, problem/solution, social proof, form, thank-you page
  • Placeholders for: business name, colours, headline, offer, form fields
  • Pre-wired: Meta Pixel, Matomo tracking, form handler stub
  • To use: duplicate folder, find-and-replace placeholders, deploy
  • ☐ 2.2 Form handler template
  • Create a reusable /api/lead.js that accepts config via environment variables
  • ENV vars: CLIENTNAME, MAUTICTAG, SLACKCHANNELID
  • Handles: Mautic contact creation, Slack notification, response
  • To use: set 3 env vars in Vercel, done
  • ☐ 2.3 Email sequence templates
  • Write generic versions of each business type drip:
  • Service business (audit/consultation offer)
  • E-commerce (discount/launch offer)
  • Professional services (guide/resource offer)
  • Store in /01-Nunya Bunya/Templates/email-sequences/
  • To use: copy, customise subject lines and body, load into Mautic
  • ☐ 2.4 Client onboarding script (v1)
  • Python script: setupnewclient.py
  • Input: client name, business type, brand colours, Slack channel ID
  • Creates: Mautic segment + emails + campaign via API
  • Output: segment ID, email IDs, campaign ID (all logged)
  • Does NOT create landing pages yet (Phase 3)

PHASE 3: Automation Agent (Day 3-4)

Time: 4-6 hours

What: Build the one-click client setup that generates everything.

  • ☐ 3.1 Expand setupnewclient.py to full agent
  • Input: client name, type, colours, domain, offer headline, lead magnet title
  • Auto-generates:
  • Mautic segment, emails (from template), campaign
  • Landing page HTML (from template, with branding applied)
  • Thank-you page HTML
  • Form handler with correct env vars
  • Shlink tracking links
  • Slack channel (via Slack API)
  • Matomo site (via Matomo API)
  • Ad copy (2 Meta, 2 Google -- from templates)
  • Output: a deployable folder + a summary of all IDs and links
  • ☐ 3.2 One-command deploy
  • npx vercel --prod from the generated folder
  • Or: push to a git repo that auto-deploys via Vercel
  • ☐ 3.3 Test with a fake client
  • Run the agent with test data
  • Verify everything works end-to-end
  • Time it -- goal is under 30 minutes from input to live

PHASE 4: Monthly Automation (Day 5)

Time: 3-4 hours

What: Automate the recurring monthly tasks so they don't eat your time.

  • ☐ 4.1 Automated lead alerts (already mostly done)
  • Verify n8n workflows are active:
  • Form submission --> Slack notification
  • Mautic email opened --> Slack notification (optional, can be noisy)
  • Hot lead (score >= 50) --> urgent Slack alert
  • Test each workflow
  • ☐ 4.2 Automated monthly reports
  • Build a Metabase dashboard template for clients:
  • New leads this month
  • Email open rate / click rate
  • Landing page conversion rate
  • Ad spend and cost per lead (manual input or connected to Meta API)
  • Schedule Metabase to email the dashboard as PDF on the 1st of each month
  • Or: create a shareable dashboard link for each client
  • ☐ 4.3 Ad performance monitoring
  • Set up n8n workflow: weekly check of Meta Ads API
  • Alert to Slack if cost-per-lead exceeds threshold
  • Alert if ad spend is $0 (ads paused or billing issue)
  • ☐ 4.4 Email health monitoring
  • n8n workflow: daily check of Mautic bounce rate
  • Alert if bounce rate exceeds 5% (list quality issue)
  • Alert if any campaign has 0 sends in 24 hours (something broken)

PHASE 5: Sales Pipeline (Day 6)

Time: 2-3 hours

What: Build the system to actually get and close clients.

  • ☐ 5.1 Nunya Bunya's own funnel is live and working
  • nunyabunya.com/free-audit --> form --> Mautic drip --> booking
  • nunyabunya.com/checklist --> form --> Mautic drip --> booking
  • Both funnels tested end-to-end
  • ☐ 5.2 Proposal template
  • Create or update the proposal template (PandaDoc or PDF)
  • Include: scope, deliverables, timeline, pricing, terms
  • Pre-filled with standard Nunya Bunya packages
  • ☐ 5.3 Contract template
  • Review existing CLIENT CONTRACT.md
  • Make sure it covers: scope, payment terms, cancellation, IP rights
  • Ready to send digitally (PandaDoc or DocuSign)
  • ☐ 5.4 Outreach system
  • Daily: post in 3 Facebook groups (free advice, soft CTA)
  • Daily: engage on 5 LinkedIn posts
  • Weekly: 2 cold outreach DMs to businesses that need help
  • Weekly: check Google Maps for new Brisbane businesses without marketing
  • Track all outreach in Twenty CRM
