STEP 1: SERVER SETUP

Everything runs on a single Hetzner Cloud VPS. Here's how to get one running in 10 minutes.

1.1Create a Hetzner Account

Go to hetzner.com/cloud and sign up. Create a new project, then spin up a server:

  • Location: Choose closest to your audience (Nuremberg for EU, Ashburn for US)
  • Image: Ubuntu 22.04
  • Type: CX21 (2 vCPU, 4GB RAM) — $10/mo is plenty to start
  • Add your SSH key for secure access

1.2Install Docker & Docker Compose

SSH into your server and run:

# Update system apt update && apt upgrade -y # Install Docker curl -fsSL https://get.docker.com | sh # Install Docker Compose apt install docker-compose-plugin -y # Verify docker --version docker compose version

1.3Set Up Traefik (Reverse Proxy + Auto-SSL)

Traefik handles routing and auto-provisions SSL certificates for all your subdomains:

mkdir -p /opt/traefik && cd /opt/traefik touch acme.json && chmod 600 acme.json

Create docker-compose.yml:

version: "3.8" services: traefik: image: traefik:v2.10 command: - "--providers.docker=true" - "--providers.docker.exposedbydefault=false" - "--entrypoints.web.address=:80" - "--entrypoints.websecure.address=:443" - "--certificatesresolvers.le.acme.httpchallenge=true" - "--certificatesresolvers.le.acme.httpchallenge.entrypoint=web" - "--certificatesresolvers.le.acme.email=you@yourdomain.com" - "--certificatesresolvers.le.acme.storage=/acme.json" - "--entrypoints.web.http.redirections.entryPoint.to=websecure" ports: ["80:80", "443:443"] volumes: - /var/run/docker.sock:/var/run/docker.sock:ro - ./acme.json:/acme.json networks: [web] restart: always networks: web: external: true
# Create shared network and start Traefik docker network create web && docker compose up -d

STEP 2: DNS & DOMAINS

Point subdomains to your server so each tool gets its own clean URL.

In your domain registrar (GoDaddy, Namecheap, Cloudflare, etc.), add A records pointing to your server's IP:

Subdomain Tool Type Value
crm.yourdomain.comTwenty CRMAYOUR_SERVER_IP
mautic.yourdomain.comMauticAYOUR_SERVER_IP
analytics.yourdomain.comMatomoAYOUR_SERVER_IP
reports.yourdomain.comMetabaseAYOUR_SERVER_IP
go.yourdomain.comShlinkAYOUR_SERVER_IP
auto.yourdomain.comn8nAYOUR_SERVER_IP
cal.yourdomain.comCal.comAYOUR_SERVER_IP
Tip: DNS propagation can take up to 48 hours, but usually takes 5–30 minutes. Don't start installing tools until your subdomains resolve. Use ping crm.yourdomain.com to check.

STEP 3: INSTALL YOUR TOOLS

Each tool gets its own Docker Compose file. Copy-paste these configs and you're running.

Mautic — Email Marketing & Automation

Replaces: Mailchimp, ActiveCampaign, ConvertKit

Open-source marketing automation. Email campaigns, drip sequences, lead scoring, forms, and segmentation. Unlimited contacts, unlimited emails.

mkdir -p /opt/mautic && cd /opt/mautic

Create docker-compose.yml:

version: "3.8" services: mautic: image: mautic/mautic:latest environment: - MAUTIC_DB_HOST=mautic-db - MAUTIC_DB_USER=mautic - MAUTIC_DB_PASSWORD=CHANGE_THIS_PASSWORD - MAUTIC_DB_NAME=mautic volumes: [mautic_data:/var/www/html] labels: - "traefik.enable=true" - "traefik.http.routers.mautic.rule=Host(`mautic.yourdomain.com`)" - "traefik.http.routers.mautic.tls.certresolver=le" networks: [web, internal] restart: always mautic-db: image: mysql:8.0 environment: - MYSQL_ROOT_PASSWORD=CHANGE_THIS_PASSWORD - MYSQL_DATABASE=mautic - MYSQL_USER=mautic - MYSQL_PASSWORD=CHANGE_THIS_PASSWORD volumes: [mautic_db:/var/lib/mysql] networks: [internal] restart: always volumes: mautic_data: mautic_db: networks: web: { external: true } internal:
docker compose up -d

Visit mautic.yourdomain.com and complete the setup wizard.

STEP 4: CONNECT THE PIPELINE

Individual tools are useful. Connected tools are powerful. Here's how to wire everything together.

Visitor lands on your website
↓
Matomo tracks the visit (pages, referrer, time on site)
↓
Visitor fills out a form or downloads a lead magnet
↓
Mautic captures their email, adds tags, starts a drip campaign
↓
Twenty CRM creates a contact record with lead score
↓
n8n sends a Slack notification to your team
↓
Lead books a call via Cal.com
↓
Track the deal in Twenty CRM, report results in Metabase

Key Integrations

  • Website → Mautic: Add the Mautic tracking pixel to your site. Create Mautic forms or connect existing forms via API.
  • Mautic → Twenty CRM: Use n8n to sync new Mautic contacts into Twenty CRM automatically.
  • Mautic → Slack: Use n8n to post a notification when a new lead comes in.
  • Matomo → Metabase: Connect Metabase to Matomo's MySQL database for custom dashboards.
  • Shlink → Campaigns: Create tracked short links for every email, social post, and ad.
Cal.com note: Cal.com's self-hosted setup is more involved than the others. You can use their free hosted tier at cal.com to skip the Docker setup. We use the hosted version ourselves.

STEP 5: MAINTAIN YOUR STACK

Your stack is running. Here's how to keep it healthy.

Updates

Pull the latest images and restart. Do this monthly:

# Update everything at once: for dir in /opt/traefik /opt/mautic /opt/twenty /opt/matomo /opt/metabase /opt/shlink /opt/n8n; do cd "$dir" && docker compose pull && docker compose up -d done

Backups

Back up your Docker volumes regularly with a cron job:

# Add to crontab (crontab -e) — runs daily at 3am 0 3 * * * docker run --rm \ -v mautic_db:/data -v /backups:/backup alpine \ tar czf /backup/mautic-db-$(date +\%Y\%m\%d).tar.gz /data

Monitoring

  • Check disk space monthly: df -h
  • Monitor container health: docker ps
  • Set up UptimeRobot (free for 50 monitors) for uptime alerts
  • Keep your server OS updated: apt update && apt upgrade -y
Scaling: The CX21 ($10/mo) handles most small businesses easily. If you hit limits, upgrade to CX31 (4 vCPU, 8GB, $18/mo) with one click. No downtime.
